# Why constraints.txt instead of pinning here:
#   Airflow 2.8.1 ships its own constraint file per Python version.
#   Hard-pinning conflicts with it on macOS Arm64 (M-series) and some
#   Windows/conda environments. The constraint file bounds the dangerous
#   packages (pyarrow, numpy) while letting the resolver handle the rest.
#   pyarrow IS hard-pinned in constraints.txt because Parquet schema
#   encoding changed between major versions — all team members must match.
# =============================================================================
